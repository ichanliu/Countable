# Countable - Project Guide

## Overview
Countable is a React Native + Expo app for tracking countdown/count-up events.
Built with Expo SDK 54, TypeScript, Expo Router (file-based routing), and a custom 
Android Widget module (Expo local module).

## Project Structure

countable/

## Getting Started

### Prerequisites
- Node.js 18+
- Expo CLI: npx expo --version

### Install & Run
npm install
npx expo start            # Development
npx expo run:android      # Native debug build (requires Android SDK)

### Build Android APK (EAS Cloud)
npx eas build --platform android --profile preview

Note: EAS Hobby plan = 15 builds/month. Used ~10 already.

## Key Technical Details

### Android Widget
- Located in modules/countdown-widget/ as an Expo local module
- Kotlin + RemoteViews, auto-linked via expo-modules.config.js
- Displays pinned event's count, label, color, and background image
- Click -> deep link: countdowns://event-detail?eventId=xxx
- Background image: stored in SharedPreferences, displayed via setImageViewUri()

### Event Data Model
interface CountdownEvent {
  id: string;
  title: string;
  targetDate: string;       // ISO date
  imageUri?: string;        // Card preview image
  bgImageUri?: string;      // Detail page full-screen background
  widgetImageUri?: string;  // Widget background (falls back to imageUri)
  isPinned: boolean;        // Starred/pinned -> appears at top
  createdAt: string;        // ISO date
}

### Event Detail Page (event-detail.tsx)
- Two-page snap scroll (pagingEnabled)
  - Page 1: Full-screen hero with bg image + centered countdown
  - Page 2: Progress bar + edit controls (bg, widget, target, created)
- Gradient overlay: ['transparent', 'rgba(0,0,0,0.3)', 'rgba(0,0,0,0.5)']
- Progress = elapsed / total days (only for future/today events)

### Navigation
- Home -> tap card -> event-detail?eventId=xxx
- Home -> settings gear -> /settings
- Home -> FAB -> /add-event
- Widget click -> deep link event-detail?eventId=xxx

### Known Constraints
- Local build requires 8GB+ RAM (C++ compilation)
- Use EAS cloud builds for release APKs
- react-native-svg v15.x incompatible with RN 0.81.5 New Architecture
- Progress bar uses horizontal bar (no SVG) to avoid C++ dependency
